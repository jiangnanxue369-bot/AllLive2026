# GitHub 编译指南

本指南将帮助您在 GitHub 上使用 Actions 编译 AllLive 项目。

## 准备工作

1. **Fork 仓库**：首先，在 GitHub 上 fork 此仓库到您自己的账户。

2. **启用 GitHub Actions**：确保您的 fork 仓库中已启用 GitHub Actions。

## 编译步骤

1. **推送代码**：将修改后的代码推送到您的 fork 仓库的 `master` 分支。

2. **触发编译**：
   - 推送代码后，GitHub Actions 会自动触发编译流程。
   - 您也可以手动触发编译：进入仓库的 "Actions" 标签页，选择 "Build UWP App" 工作流，然后点击 "Run workflow" 按钮。

3. **等待编译完成**：编译过程可能需要 5-10 分钟，取决于 GitHub 的服务器负载。

4. **下载构建产物**：编译完成后，您可以在 "Actions" 标签页中找到对应的工作流运行，然后下载 "AllLive-UWP-Package" 构建产物。

## 构建产物说明

构建产物包含以下文件：

- **AllLive.UWP_版本号_x64.appx**：UWP 应用安装包
- **AllLive.UWP_版本号_x64.cer**：应用签名证书
- **AllLive.UWP_版本号_x64.appxsym**：符号文件

## 安装说明

1. **安装证书**：
   - 双击 `.cer` 文件，按照提示安装证书到 "本地计算机" → "受信任的根证书颁发机构"。

2. **安装应用**：
   - 双击 `.appx` 文件，按照提示完成安装。

## 注意事项

1. **证书问题**：由于使用的是临时生成的自签名证书，您需要在安装前手动信任该证书。

2. **平台限制**：构建产物仅包含 x64 版本的应用，适用于 64 位 Windows 10 或 Windows 11。

3. **系统要求**：应用需要 Windows 10 1903 或更高版本。

4. **依赖项**：GitHub Actions 工作流会自动处理所有 NuGet 依赖项。

## 故障排除

如果编译失败，请检查以下几点：

1. **工作流文件**：确保 `.github/workflows/build.yml` 文件存在且配置正确。

2. **NuGet 依赖**：如果依赖项下载失败，可能是网络问题，稍后重试即可。

3. **证书生成**：如果证书生成失败，请确保项目配置中 `GenerateTemporaryStoreCertificate` 设置为 `True`。

4. **构建日志**：查看 GitHub Actions 的构建日志，了解具体的错误信息。

## 常见问题

**Q: 安装应用时提示证书错误怎么办？**
A: 请确保您已正确安装 `.cer` 证书到 "受信任的根证书颁发机构"。

**Q: 应用无法安装在我的设备上怎么办？**
A: 请确保您的设备运行的是 Windows 10 1903 或更高版本，并且是 64 位系统。

**Q: 编译时间过长怎么办？**
A: GitHub Actions 的构建时间可能会因服务器负载而有所不同，请耐心等待。如果超过 30 分钟仍未完成，可能是遇到了问题，建议取消并重新触发编译。
